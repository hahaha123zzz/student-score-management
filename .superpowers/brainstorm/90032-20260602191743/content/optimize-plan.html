现有项目优化方案

不推倒重来，在现有代码上做减法 —— 只改你们能力范围内的部分

🎯 优化总览：3个方向

1. 前端瘦身

6个页面 → 3个页面
去掉Chart.js、复杂JS
只留表格+表单+fetch

2. 后端精简

删除enhance/extra模块
删除家长角色路由
lambda → 普通函数

3. 知识点降维

C++17 lambda → 普通函数
structured binding → 传统写法
保留CMake + httplib + json

📉 前端优化：从 6 页砍到 3 页

文件 现状 优化后 理由

parent.html 保留 ❌ 删除 课程只要2个角色

🗑️ 后端删除清单

要删除的 原因
src/enhance.cpp + include/enhance.h 积分/打卡/复习计划/目标 — 非课程要求
src/extra.cpp + include/extra.h 错题本/偏差分析/雷达图/建议 — 非课程要求
main.cpp 中所有 enhance/extra 路由 约30个路由端点，涉及复杂JSON处理
main.cpp 中家长相关路由 课程只要管理员+学生
main.cpp 中 seedData() 的额外生成 error/goals/points/checkins 数据不再需要

🔧 C++17 语法降维（让代码好读）

❌ 现在（大一难读）
auto [it, inserted] = students.emplace(si, info);
auto [rank, total] = getTotalRank(sid);

handler([&](const Request& req,
            Response& res) { ... });
✅ 优化后（大一能读）
auto result = students.emplace(si, info);
auto& it = result.first;
bool inserted = result.second;

RankResult rank = getTotalRank(sid);
int total = rank.total;

// 路由用普通函数
handleStudentList(req, res);
